# 이벤트 로거

## 문제

클로저를 활용하여 카테고리별 독립적인 로그 번호를 관리하는 로거 팩토리를 작성하세요.

1. `Action<string> CreateLogger(string category)` 메서드를 작성하세요. 이 메서드는 `Action<string>` 타입의 람다식을 반환합니다.
2. 반환되는 람다식은 호출될 때마다 내부 로그 번호를 1씩 증가시키고, `"[카테고리 #번호] 메시지"` 형식으로 콘솔에 출력합니다. 
   (번호는 1부터 시작)
3. `"시스템"` 카테고리로 로거를 하나 생성하고, `"네트워크"` 카테고리로 로거를 하나 더 생성하세요.
4. 아래 예상 실행 결과와 동일한 순서로 각 로거를 호출하여 출력하세요.
5. 두 로거의 로그 번호가 서로 독립적으로 증가하는지 확인하세요.

## 예상 실행 결과

```
[시스템 #1] 서버를 시작합니다.
[시스템 #2] 초기화가 완료되었습니다.
[네트워크 #1] 클라이언트가 접속했습니다.
[시스템 #3] 요청을 처리합니다.
[네트워크 #2] 데이터를 수신했습니다.
[네트워크 #3] 클라이언트가 종료했습니다.
```
